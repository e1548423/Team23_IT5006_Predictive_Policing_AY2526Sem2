# Spatio-Temporal Violent Crime Prediction

Predict the probability of violent crime occurring in a specific area during a specific patrol shift, and dispatch patrol units to high-risk tiles for proactive presence.

---

## Overview

This system uses historical Chicago crime data to train a binary classifier that answers: **"Will a violent crime occur in this hexagonal tile during this shift?"**

The pipeline is split into two notebooks:

| Notebook | Purpose |
|---|---|
| `Crime_Prediction_Training (Violent Crime).ipynb` | Data loading, preprocessing, feature engineering, model training, evaluation, and packaging |
| `Inference_Engine_UI.ipynb` | Load the packaged model and run interactive predictions with map visualisation |

---

## Workflow

```
[Training Notebook]                    [Inference Notebook]
  Load Data                               Load deployment/
  → Filter & Clean                        → Enter date + shift
  → Build H3 Grid                         → Score all tiles
  → Engineer Features                     → View ranked tiles + map
  → Train & Evaluate Models               → Export patrol briefing CSV
  → Package Model → deployment/
```

## Key Design Decisions

- **H3 resolution 8** (~0.7 km² hexagons) balances spatial granularity with sufficient crime density per tile for meaningful signal.
- **Shift-level prediction** rather than hourly allows operational alignment with real patrol schedules.
- **Binary target** (`crime occurred: yes/no`) is preferred over count regression given the heavily zero-inflated distribution.
- **Threshold 0.150** is set below the default 0.5 to maximise recall — it is operationally more valuable to dispatch to some false positives than to miss true crime locations.
- **Probability calibration** is applied post-training so that output probabilities reflect true empirical frequencies rather than raw model scores.
